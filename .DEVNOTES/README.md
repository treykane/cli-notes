# .DEVNOTES

Shared workspace notes for this repo. Use these files to keep context, decisions, and ongoing work discoverable.

## Index
- `README.md` — This index and conventions.
- `DEVELOPMENT.md` — Quickstart, layout, runtime flow, troubleshooting.
- `CONTRIBUTING.md` — Code style, PR expectations, contribution practices.
- `DEMO.md` — Walkthrough and feature demo script.
- `MEMORY.md` — Long-lived decisions, conventions, and historical context.
- `TASKS.md` — Current tasks and backlog.

## Conventions
- Keep notes concise and dated.
- Prefer bullets for scan-ability.
- Link to PRs/issues when applicable.
- Avoid duplicating content across files; link to the canonical doc instead.
- Treat this as living documentation.
